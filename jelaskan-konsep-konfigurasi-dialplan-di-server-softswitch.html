1. 1.bagaimana konsep kerja server softswitch?2.apakah yang dimaksud sip dan softswitch?3.jelaskan bagaimanakah prosedur instalasi server softswitch berbasis sip?


1. Softswitch merupakan suatu alat yang bekerja pada jaringan VoIP. Softswitch menghubungkan antara jaringan sirkuit yang ada dengan jaringan paket. Jaringan tersebut meliputi jaringan telpon tetap (PSTN), internet yang berbasis IP, kabel TV dan juga jaringan seluler (GSM). Lalu bagaimana cara kerja Softswitch sendiri? Pada artikel kali ini akan kita bahas Konsep Kerja Protokoler Server Softswitch yang merupakan bagian dari artikel bertema VoIP.

2. SIP adalah suatu signalling protocol pada layer aplikasi yang berfungsi untuk membangun, memodifikasi, dan mengakhiri suatu sesi multimedia yang melibatkan satu atau beberapa pengguna. Sesi multimedia adalah pertukaran data antar pengguna yangbisa meliputi suara, video, dan text.

3. Pertama adalah menghubungkan SIP proxy dengan PBX system existing sebagai gateway ke PSTN. Untuk menghubungkan dari IP Network ke PSTN, SIP proxy akan me-routing-kan seluruh panggilan dari panggilan VoIP ke PSTN. Link koneksi bisa menggunakan PRI ISDN maupun T1 Pertama yang harus dilakukan adalah setup SIP proxy (SIP call manager), setelah set-up berjalan dengan baik, selanjutnya tinggal menghubungkan ke PBX existing dengan link ISDN, E1 atau T1.

Kedua ,selanjutnya adalah migrasi existing user dan services ke SIP. Migrasi semua handset ke SIP baik menggunakan gateway maupun mengganti dengan handset yang native SIP. Pendekatan dengan menggabungkan keduanya biasanya menjadi pilihan yang baik. Selanjutnya adalah set-up SIP application server untuk mengakomodasi fitur-fitur yang dibutuhkan user seperti voice mail, multi-party conferencing, IVR, fine me follow me dan masih banyak lagi fitur yang disupport SIP.
Ketiga, Terakhir melakukan routing panggilan ke luar melalui koneksi internet.Banyak operator memulai menjual SIP based dengan menawarkan layanan long distance dengan harga yang lebih murah. Yang perlu diantisipasi adalah nature network internet yang unpredictable (seperti jitter dan error rate) dan tentu saja security. Sebagai tahap awal SIP dapat digunakan sebagai alternatif untuk melakukan panggilan ketika diluar kantor atau teleworkers.

2. Jelaskan secara singkat konsep kerja server softswitch


Softswitch adalah sekumpulan aplikasi dan perangkat protokol internet yang berfungsi mendukung kinerja perangkat lain agar dapat mengakses layanan internet maupun telekomunikasi secara umum dalam sebuah jaringan IO.

Pembahasan

Pada kesempatan ini, soal meminta kita untuk menyajikan konsep kerja server softswitch. Berikut kakak akan mencoba menjawab pertanyaan tersebut.

Server Softswitch bekerja dengan beberapa langkah atau tahapan, yaitu sebagai berikut:

Perangkat telepon VoIP dan gateway saling berkirim sinyal satu sama lain dalma sebuah jaringan yang sama .Ketika seluruh perangkat tersebut menerima sinyal, maka server Softswitch akan menjalankan tugasnya dengan mengenali panggilan yang masuk berdasarkan sumbernya, yakni antara jaringan IP atau pesawat telepon biasa. Jika sinyal berasal dari jaringan IP, server Softswitch akan memberikan perintah kepada perangkat pemberi sinyal dan penerima untuk melangsungkan komunikasi. Pada saat yang bersamaan, softswitch turut bekerja dalam mengontrol stabilitas panggilan yang dilakukan melalui jaringan internet yang digunakan.

5. 5. Jelaskan arsitektur utama dari server softswitch!​


Arsitektur utama Softswitch, yaitu:

Media GatewayMedia Gateway Controller (MGC) atau Call AgentMedia ServerApplication serverOperating support system (OSS)

Pembahasan

Softswitch adalah istilah kolektif untuk pendekatan baru untuk teknologi switching yang mencakup istilah kontrol panggilan dan pemrosesan panggilan.

Berikut penjelasan dari arsitektur utama dari server softswitch:

Media Gateway (MG) bertindak sebagai elemen transport yang mengarahkan lalu lintas di jaringan softswitch, mengirim dan menerima lalu lintas dari jaringan lain yang berbeda seperti PSTN, PLMN, VoIP H.323, dan jaringan akses pelanggan. .Media Gateway Controller (MGC) atau Call Agent, adalah elemen utama dari soft switch, mengendalikan semua layanan dan sesi komunikasi, mengkoordinasikan interaksi elemen jaringan lain, PSTN, SS7, dan jaringan IP. Kemampuan menjembatani jaringan dengan karakteristik yang berbeda saling berhubungan menggunakan protokol SIP-T.Signaling Gateway (SG) menciptakan jembatan antara jaringan SS7 dan jaringan IP di bawah kendali MGC. SG hanya menangani pensinyalan SS7 dan MGC menangani sirkuit suara yang diatur oleh mekanisme pensinyalan SS7.Server media memenuhi fungsi itu. Dengan kata lain, ini meningkatkan soft switch dengan fungsionalitas media. Mendukung pemrosesan sinyal digital (DSP) jika diinginkan.Server aplikasi adalah elemen jaringan yang menyediakan aplikasi tambahan di luar fungsionalitas telepon yang memerlukan server terpisah. Contoh: pesan suara, panggilan prabayar, SMS darat, VPN suara, dan sebagainya.Operating support system (OSS) adalah elemen jaringan yang dirancang untuk mendukung pengoperasian dan pemeliharaan jaringan. Manajemen jaringan, penyediaan, penagihan, pemantauan, statistik, dan sebagainya.

8. Bagaimana cara kerja server softswitch berkaitan dengan pbx


Cara kerja Server Softwitch berkaitan dnegan PBX yaitu :

Sebuah sistem IP PBX terdiri dari satu atau lebih telepon SIP, server IP PBX dan cara opsional VOIP Gateway untuk terhubung ke jalur server, klien SIP, baik berupa perangkat lunak atau perangkat keras berbasis ponsel, mendaftar ke server IP PBX, dan ketika mereka ingin membuat panggilan maka meminta IP PBX untuk melakukan panggilan. IP PBX memiliki daftar semua ponsel atau pengguna dan alamat sesuai dengan SIP dan dengan demikian dapat menghubungkan panggilan internal atau rute panggilan eksternal baik melalui Gateway VOIP atau menyediakan layanan VOIP.

Pembahasan

PBX (Private Branch Exchange) adalah sebuah sentral privat dengan fitur seperti sentral public yang digunakan oleh suatu lembaga / perusahaan dalam melayani komunikasi internal perusahaan tersebut.VOIP (Voice Over IP) adalah teknologi yang memungkinkan percakapan suara jarak jauh melalui media internet. Data suara diubah menjadi kode digital dan dialirkan melalui jaringan yang mengirimkan paket data dan bukan lewat jalur sirkuit analog biasa.SIP (Session Intiation Protocol) adalah suatu signalling protokol pada layer aplikasi untuk membangun, memodifikasi, dan mengakhiri suatu sesi multimedia yang melibatkan satu atau beberapa pengguna.

1. Konfigurasi Ekstensi Server Softwitch

Untuk melakukan konfigurasi ekstensi pada server softswitch, kita harus melakukakn pengeditan pada file sip.conf.

Ada dua hal yang harus dikonfigurasi pada file itu, yaitu konfigurasi user umum dan konfigurasi user data atau extension.

Konfigurasi Data Account Umum :

[general] -->> context umum,harus ada

context=tkj-->> nama context user

port=5060 -->> default port untuk SIP

binaddr=0.0.0.0 -->> listen semua ip_addr yg request

srvlookup=yes

tos=0x18

videosupport=yes

Konfigurasi Data Account User / Extensions :

;softphone (nama atau nomor account)

[101] (user/extension)

type=friend (tipe account)

username=101 (login account)

secret=101 (password account)

host=dynamic (host yang menjadi IP PBX,dapat berubah)

nat=no (tanpa NAT)

dtmfmode=rfc2833 (RTP Payload for DTMF Digits, Telephony Tones and Telephony Signals)

allow=all (mode codec, bisa semua jenis codec)

callerid=”sip00” (id client)

context=tkj (context jaringan user)

canreinvite=no (mekanisme canreinvite)

mailbox=101@tkj (username@contex)

2. Konfigurasi Dial Plan Server Softwitch

Untuk mengkonfigurasi dial plan, edit file extensions.conf dengan mengetik nano /etc/asterisk/extensions.conf. Pastikan bahwa seluruh perintah pada file ini sudah dinon aktifkan. Ketik perintah di bawah ini pada bagian paling akhir dari isi file extensions.conf.

Dial antar ekstensi pada IP-PBX [tkj] (seluruh dial plan di bawah hanya berlaku bagi context "TKJ"

exten =>101,1,Dial(SIP/101,20) --> Dial ext 101 dengan protokol SIP, time out 20 detik

exten =>101,2,Hangup -->> setelah timeout 20 detik dilakukan hangup

exten =>102,1,Dial(SIP/102,20)

exten =>102,2,Hangup

Jawaban dan penjelasan:

Server softswitch adalah sebuah sistem komunikasi yang digunakan dalam layanan VoIP (Voice over Internet Protocol) yang digunakan untuk mengarahkan panggilan suara dari satu titik ke titik lainnya melalui jaringan IP. Berikut adalah beberapa peralatan yang digunakan oleh server softswitch:

Server hardware: Digunakan untuk menjalankan software softswitch dan mengelola jaringan IP.
Softswitch software: Merupakan program yang digunakan untuk mengontrol panggilan suara melalui jaringan IP.
Session Border Controller (SBC): Digunakan untuk memperkuat keamanan jaringan IP dan mengontrol lalu lintas panggilan suara.
Media Gateway (MGW): Digunakan untuk mengubah sinyal suara dari format TDM (Time Division Multiplexing) ke IP.
Signaling Gateway (SG): Digunakan untuk menghubungkan protokol yang berbeda di jaringan IP.
Billing System: Digunakan untuk menghitung biaya panggilan dan menyimpan data panggilan.
Network Management System (NMS): Digunakan untuk mengelola dan memantau kinerja jaringan IP dan peralatan yang digunakan.
Database server: Digunakan untuk menyimpan data pelanggan dan konfigurasi jaringan IP.
Softphone: Merupakan perangkat lunak yang digunakan untuk membuat dan menerima panggilan suara melalui jaringan IP.
IP Phone: Merupakan perangkat keras yang digunakan untuk membuat dan menerima panggilan suara melalui jaringan IP.
Firewall: Digunakan untuk melindungi jaringan IP dari serangan dan ancaman keamanan.
Router dan switch: Digunakan untuk mengarahkan lalu lintas data dan suara melalui jaringan IP.
SIP Trunk: Digunakan untuk menghubungkan jaringan IP dengan jaringan PSTN (Public Switched Telephone Network).
VoIP Gateway: Digunakan untuk menghubungkan jaringan IP dengan jaringan PSTN atau ISDN (Integrated Services Digital Network).

Peralatan tersebut di atas adalah beberapa peralatan yang digunakan dalam server softswitch. Namun, setiap vendor softswitch mungkin memiliki konfigurasi peralatan yang berbeda tergantung pada kebutuhan dan spesifikasi pelanggan.

18. 4. Berdasarkan hasil analisis dan informasi yang telah kamu temukan, gambarkan topologi ekstensi dan dial plan server softswitch beserta cara kerjanya! Jelaskan secara rinci! Topologi ekstensi dan dial plan server softswitch:​


Jawaban:

Untuk menggambarkan topologi ekstensi dan dial plan server softswitch, kita harus memahami dua komponen utama dalam arsitektur softswitch: server ekstensi dan server dial plan.

Server ekstensi adalah komponen yang bertanggung jawab untuk mengkoordinasikan ekstensi telepon yang digunakan oleh pengguna. Ekstensi telepon adalah nomor yang digunakan oleh pengguna untuk menghubungi satu sama lain atau untuk menghubungi sistem telepon lainnya. Server ekstensi dapat diimplementasikan sebagai aplikasi berbasis software atau sebagai perangkat keras yang terpisah.

Server dial plan adalah komponen yang bertanggung jawab untuk mengkoordinasikan routing panggilan telepon. Dial plan adalah aturan yang digunakan untuk menentukan di mana panggilan telepon harus diarahkan berdasarkan nomor yang ditelpon. Server dial plan dapat diimplementasikan sebagai aplikasi berbasis software atau sebagai perangkat keras yang terpisah.

Secara umum, topologi ekstensi dan dial plan server softswitch terdiri dari beberapa komponen utama, yaitu:

   Server ekstensi dan dial plan

   Gateway PSTN/PBX

   Jaringan IP atau VoIP

Server ekstensi dan dial plan dapat terhubung ke gateway PSTN/PBX melalui jaringan IP atau VoIP. Gateway PSTN/PBX digunakan untuk mengubah sinyal telepon analog menjadi sinyal digital yang dapat diteruskan melalui jaringan IP. Jaringan IP atau VoIP digunakan untuk mengirimkan sinyal telepon digital dari server ekstensi dan dial plan ke gateway PSTN/PBX.

Cara kerja dari topologi ekstensi dan dial plan server softswitch adalah sebagai berikut:

   Pengguna menghubungi ekstensi telepon yang ditentukan melalui telepon IP atau telepon analog

   Sinyal telepon diterima oleh server ekstensi dan dial plan

   Server ekstensi dan dial plan menentukan tujuan panggilan berdasarkan aturan dial plan yang telah ditentukan

   Server ekstensi dan dial plan mengirimkan sinyal telepon ke gateway PSTN/PBX

   Gateway PSTN/PBX meneruskan sinyal telepon ke tujuan yang ditentukan

   Panggilan telepon diterima oleh pengguna tujuan

19. jelaskan konsep konfigurasi dial plan di server softwitch


Konsep dari konfigurasi dial plan di dalam server softswitch adalah mengedit ekstensi dengan cara mengetik nano pada asterisk dial plan. Ketik perintah “Dial” pada bagian akhir dari file extensions.conf. Setelah itu tunggu selama beberapa detik. Setelah berhasil melakukan dial lakukan hangup, maka konfigurasi telah selesai dilakukan. Jangan lupa sebelum melakukan konfigurasi nonaktifkan dulu semua perintah dalam file ekstensi.

Pembahasan:

Dial plan adalah suatu program yang mengontrol adanya panggilan yang masuk atau keluar. Bukan hanya panggilan dial plan juga mampu menghubungkan antara jaringan sirkuit dengan jaringan paket seperti jaringan telepon tetap (PSTN), internet yang berbasis IP, TV kabel, serta jaringan seluler yang telah ada.  Dial plan mampu merutekan panggilan dari satu server ke server lain tanpa adanya gangguan. Hal ini dikarenakan dial plan mampu mengatur keluar masuknya server di dalam sambungan. Banyak sekali server yang ada saat ini, salah satu contohnya adalah server softswitch. Dial plan yang ada di dalam server softwitch akan mengatur setiap ekstensi untuk menghubungi sesama ekstensi atau dial trunk begitu juga sebaliknya. Dalam membuat dial plan kita harus menetapkan nomor telepon individu atau ekstensi masing-masing pengguna, dengan cara mengedit file extensions.conf yang terletak pada folder /etc/asterisk/.

29. jelaskan hubungan antara pbx server softswitch dengan telepon sip​


Telepon SIP adalah bagian integral dari infrastruktur PBX Server Softswitch. Adapun infrastruktur tersebut (PBX Server Softswitch) merupakan perangkat utama yang berperan untuk mengatur lalu lintas komunikasi panggilan telepon (perangkat komunikasi, bisa berbentuk telepon meja, telepon selular atau komputer yang menggunakan standar SIP) dari satu saluran ke saluran telepon yang lain.

Pembahasan

Telepon SIP

Sederhananya, telepon SIP merupakan perangkat dengan standar SIP dalam mengatur setiap panggilan telepon. Alih-alih menggunakan jaringan telepon tetap (PSTN) seperti Telkom, telepon SIP membawa seluruh pertukaran suara melalui jaringan berbasis IP yang lebih populer atau kita kenal dengan protokol VoIP (voice over internet protocol)

Telepon SIP sendiri bisa berbentuk perangkat keras yang menyerupai telepon pada umumnya, sedangkan yang lainnya bisa berbentuk sebagai perangkat lunak yang terpasang pada komputer/smartphone.

PBX Server Softswitch

Secara teknis PBX Server Softswitch berfungsi untuk mengatur lalu lintas komunikasi antar saluran telepon atau perangkat yang terdiri dari telepon bebasis SIP, dimana setiap perangkat telepon tersebut harus terdaftar di PBX Server Softswitch itu sendiri.
