Analisis Kebutuhan Jaringan Komputer untuk perangkat server dan client

Kebutuhan Hardware :

1.    Personal computer
Personal Computer  atau PC merupakan perangkat utama dalam suatu jaringan computer. PC inilah yang akan bekarja mengirim dan mengakses data dalam jaringan.

2.    NIC 
NIC merupakan port yang menghubungkan computer dengan jaringan. Port atau card ini bergabung di dalam PC yang menempel pada Main Board. Berdasarkan tipe bus, ada beberapa tipe NIC atau Network Card, yaitu ISA dan PCI.

3. Kabel jaringan
Kabel jaringan merupakan media yang digunakan untuk menghubungkan kopmputer satu dan komputer lainnya yang terhubung ke jaringan. beberapa jenis kabel jaringan  adalah sebagai berikut:

Kabel Coaxial, Tampilan fisik terdiri dari kawat tembaga sebagai inti yang dilapisi oleh isolator.Fiber Optik, Terbuat dari serat kaca dengan teknologi canggih dan mempunyai kecepatan transfer data yang lebih cepat.Twisted Pair, Terdiri dari beberapa kabel yang saling melilit. Ada dua jenis kabel Twisted Pair. Shielded Twisted Pair (STP) dan Unshielded Twisted Pair (UTP).
4. Konektor
Sebagai sarana penghubung antar kabel dengan colokan NIC.

5. Hub/Switch
Hub adalah alat yang digunakan untuk menghubungkan antar komputer dan perlatan lain dalam jaringan. Fungsi hub yaitu untuk mengarahkan informasi dalam jaringan sehingga sampai ke tujuan. Namun untuk mendapatkan kualitas transformasi data yang baik dapat digunakan switch untuk menggantikan hub. Hub dan switch memiliki persamaan fungsi untuk mengarahkan informasi dalam jaringan. Namun switch memiliki nilai lebih dibandingkan hub.

6. Modem
Modem atau Modul the Modulator adalah peralatan jaringan yang digunakan untuk terhubung ke jaringan internet menggunakan kabel telpon. Dengan kata lain, modem adalah alat yang bertugas untuk mengubah data dari bentuk digital ke analog dan sebaliknya yang bisa dipahami oleh kita.

7. Bridge
Bridge adalah peralatan yang digunakan untuk memperluas atau memecah jaringan. Bridge berfungsi untuk menghubungkan dan menggabungkan media jaringan yang tidak sama.

8. Router
Router memiliki kemampuan melewatkan paket IP dari satu jaringan yang berbeda ke jaringan lain yang bukan susunan atau jaringan yang mungkin memiliki banyak jalur diantara keduanya. Router berfungsi sebagai penghubung antar dua atau lebih jaringan untuk meneruskan data dari satu jaringan ke jaringan lainnya.

9. Repeater
Repeater berguna untuk memperkuat sinyal dengan cara menerima sinyal dari suatu segmen kabel LAN lalu memancarkan kembali dengan kekuatan yang sama dengan sinyal asli pada segmen kabel lain.

Kebutuhan Software :

1. System Operasi
Sistem operasi (bahasa Inggris: operating system ; OS) adalah seperangkat program yang mengelola sumber daya perangkat keras komputer, dan menyediakan layanan umum untuk aplikasi perangkat lunak. Contohnya, Windows, Linux, Mac OS, dll.

2. Web Server
Web Server adalah komputer server yang menyediakan informasi yang bisa dilihat oleh pengguna internet. Fungsi utama sebuah server web adalah untuk mentransfer berkas atas permintaan pengguna melalui protokol komunikasi yang telah ditentukan.

3. Web Browser
Web Browser atau dalam bahasa indonesia peramban web merupakan software yang mempunyai fungsi menampilkan halaman sebuah website. Sedangkan pengertian web browser menurut wikipedia "perangkat lunak yang berfungsi menampilkan dan melakukan interaksi dengan dokumen-dokumen yang disediakan oleh server web" .

4. File Transfer Protocol
Protokol pengiriman berkas (Bahasa inggris: File Transfer Protocol) adalah sebuah protokol Internet yang berjalan di dalam lapisan aplikasi yang merupakan standar untuk pengiriman berkas (file) komputer antar mesin-mesin dalam sebuah Antarjaringan.

Pada dasarnya yang dinamakan dengan DNS merupakan bagian dari infrastruktur suatu web. Biasanya dalam DNS tersebut akan menyediakan tata cara penerjemahan domain hingga menjadi alamat IP. Nantinya untuk melakukan setting DNS ubuntu server akan memerlukan beberapa langkah mudah.

1. Membuka File

Langkah pertama untuk melakukan setting DNS ubuntu server yaitu dengan membuka file yang diperlukan. Perlu diketahui bahwa file yang diperlukan ini masih ada, namun mungkin bentuknya akan menjadi symlink. Yang diketahui dilakukan pengendalian oleh layanan lainnya.

Untuk itulah tak diperbolehkan untuk dilakukan pengeditan dengan cara manual. Istilah yang namanya systemd-resolved merupakan suatu layanan yang diketahui memberikan resolusi DNS Name. Biasanya diperuntukan bagi aplikasi lokal maupun layanan yang mampu dikonfigurasi memakai Netplan.

2. Menyimpan di Directory

Tahap berikutnya dalam melakukan pengaturan pada DNS Ubuntu server yakni melakukan penyimpanan atas file konfigurasi dari Netplan. Kemungkinan pengguna nantinya akan menjumpai satu ataupun dua jenis file YAML pada bagian directory ini.

Kemudian bisa saja terjadi perbedaan atas nama file dari satu pengaturan dengan pengaturan lainnya. Umumnya file ini mempunyai nama tersendiri yang kemungkinan pada sistem yang dimiliki oleh pengguna akan berbeda.

Dengan file ini pengguna berkemungkinan untuk melakukan konfigurasi interface jaringan. Hal ini juga mencakup tentang gateway kemudian alamat IP, lalu DNS nameserver dan lainnya. Nantinya untuk melakukan pembukaan terhadap konfigurasi ini bisa menggunakan teks editor.

3. Mengubah IP Menjadi DNS Server Pilihan

Perlu diketahui bahwa untuk melakukan konfigurasi DNS ini pengguna perlu melakukan perubahan terhadap alamat IP sekarang ini. Hingga alamat IP tersebut menjadi DNS server yang menjadi pilihan. Contohnya ketika pengguna ingin mengubah DNS server cloudflare.

Jika demikian maka pengguna perlu melakukan perubahan pada bagian baris address yang ada. Dalam penulisan yang dipakai, DNS server perlu dipisah dengan koma. Nantinya dalam hal ini pengguna dapat melakukan penambahan dua name server lebih.

4. Mengedit File

Ketika nantinya tidak didapati entri, maka dapat dilakukan penambahan pada bagian bawah dari blok nama interface. Di waktu melakukan pengeditan file YAML, pengguna perlu memastikan bahwasanya sudah memenuhi standar indentasi dari kode YAML itu sendiri.

Apabila kemudian didapati kesalahan dalam melakukan konfigurasi, maka Netplan tak akan melakukan parsing file. Jika sudah selesai tinggal simpan saja file kemudian terapkan perubahan yang dilakukan.

5. Menghasilkan Konfigurasi

Jika sejumlah langkah sebelumnya sudah dijalankan dengan baik. Nantinya Netplan akan menghadirkan yang namanya file konfigurasi bagi layanan yang dilakukan penyelesaian dengan memakai systemd.

Guna melakukan tahapan verifikasi terhadap DNS resolver bahwasanya sudah dalam keadaan diatur, maka perlu untuk menjalankan sejumlah perintah tertentu.

Dimana perintah tersebut berkemungkinan untuk mencetak banyak informasi. Ketika nantinya pengguna sudah memakai filter string, maka selanjutnya bagian outputnya akan tampak terlihat. Untuk itulah pengguna dituntut untuk memperhatikan hal yang ada dengan jeli dan teliti.

Demikian sejumlah ulasan mengenai tata cara setting DNS ubuntu server dengan tulus langkah mudah. Cara tersebut penting diketahui terlebih bagi orang yang ingin membutuhkannya untuk keperluan tertentu. Perlu diingat bahwa pengguna akan memerlukan sejumlah langkah teknis sebagaimana penjabaran di atas.

1.Apa yang dimaksud dengan kebutuhan dalam kaitannya dengan analisa kebutuha perangkat lunak server? Maksud dari kebutuhan adalah pada saat kita akan membuat suatu aplikasi menggunakan sistem klien server maka kita harus merinci secara jelas apa yang kita butuhkan dari sisi perangkat lunak. Semisal kita akan menggunakan Linux sebagai sistem operasi karena kita mencari sistem operasi yang bersifat free daan apabila kita menggunakan Windows Server kita tidak memiliki anggaran biaya untuk mebeli lisensinya.

2. Apa saja langkah-langkah yang dilakukan untuk mengetahui kebutuhan perangkat lunak server? Langkah yang perlu dilakukan adalah merinci dengan seksama kebutuhahan apa saja yang diperlukan. Semisal sistem operasi kita membutuhkan yang gratis maka menggunakan linux, atau kebutuhan programmer membutuhkan mysql maka kita pilih mysql dan sebagainya.

3. Mengapa diperlukan analisa kebutuhan dalam menentukan kebutuhan server? Hal ini diperlukan agar dalam pembuatan aplikasi dapat berjalan dengan lancar dan tidak terganggu dengan adanya perubahan-perubahan yang tidak penting yang dapat menghambat pengembangan sistem tersebut. Contohnya kita sudah menggunakan microsoft Access sebagai databasenya namun pada kenyataannya aplikasi tersebut memerlukan struktur database yang rumit dan ternyata hanya bisa dilakukan pada postgresql atau SQL Server. Sehingga diperlukan adanya pembuatan dari awal. Hal seperti ini yang akan bisa hindari apabila analisa kebutuhan server sudah kita lakukan sebelumnya.

Jadi analisa kebutuhan server sudah sepatutnya kita lakukan untuk menghindari adanya kesalahan atau perbaikan yang sebetulnya tidak perlu terjadi apabila kita sudah menentukan kebutuhan server baik dari sisi perangkat lunak maupun perangkat keras.

Perangkat lunak atau software adalah bagian dari komputer yang terdiri dari beberapa perintah di mana pengoperasiannya dilakukan melalui mesin komputer. Dengan kata lain, software adalah perangkat yang tidak punya wujud fisik. Penjelasan lebih lanjut, pengertian software adalah perangkat lunak berisi data yang diprogram atau disimpan dengan fungsi-fungsi tertentu.

Fungsi software:

• Sebagai pelengkap komputer agar dapat dioperasikan dengan baik oleh perintah-perintah tertentu.

• Mengontrol hardware yang ada pada komputer sehingga dapat digunakan dan bekerja dengan baik sesuai kebutuhan.

• Menjadi penghubung antara beberapa software dengan hardware komputer.

• Menjadi penerjemah perintah software ke dalam bahasa mesin, supaya hardware bisa menerimanya dengan baik.

• Mengidentifikasi program di dalam komputer.
