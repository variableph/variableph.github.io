18. pemberian nama layer bertujuan untuk


Pemberian nama layer di aplikasi Adobe Photoshop, After Effect maupun aplikasi-aplikasi lain yang memiliki metode layering pada dasarnya memiliki satu tujuan yaitu untuk memudahkan kita dalam mengelola setiap project, terutama pada project yang memiliki atau terdiri dari sususan layer yang kompleks (rumit dan banyak)

Pembahasan:

Apa sih layer itu? Terlepas dari apapun aplikasinya pada umumnya layer adalah tumpukan/lapisan objek yang bisa kita modifikasi sesuai dengan kebutuhan. Bentuk objeknya itu sendiri bisa bermacam-macam, tentunya akan tergantung dari perangkat lunak yang digunakan. Sebagai contoh, dalam photoshop sebuah layer bisa berupa image, shape, teks, dan layer adjustment. Berikut penjelasan ringkasnya.

Image

Dalam photoshop, layer gambar bisa kita sebut sebagai layer itu sendiri. Bisa kosong/transparan ataupun berupa objek gambar bitmap sperti jpg, png, gif ataupun yang lainnya.

Shape

Shape adalah objek gambar yang berbasis vektor dimana bentuk-bentuknya ditentukan oleh point dan path. Yang membuat layer jenis ini berbeda ada dalam hal keluwesan ukuran, dimana kita bisa merubah ukurannya tanpa ada degradasi piksel.

Type

Type adalah objek layer yang berbentuk font atau teks. Sesusai dengan bentuknya kita bisa memodifikasi hal-hal yang berkaitan dengan teks, seperti perataan, jenis font dan sebagainya.

Adjusment

Dalam photoshop kita bisa melakukan penyesuaian pada gambar dengan 2 cara, yaitu secara langsung pada objek/layernya atau dengan cara menambahkan layer adjusment. Nah, keuntungan layer adjusment ini adalah penyesuaian yang dibuat tidak bersifat permanen.

Penjelasan:

Application Layer merupakan lapisan yang pertama pada saat sebuah data mulai ditransfer, dan merupakan lapisan terakhir yang dilewati begitu komputer client menerima data tersebut.

Fungsi dari Application layer :

Application layer, sebagai pelepas data dalam sebuah jaringan dan juga penampil data dalam sebuah jaringan memiliki beberapa fungsi, seperti :

Menyajikan interface antara aplikasi dengan jaringanMengatur bagaimana sebuah aplikasi mampu untuk mengakses jaringanMembuat pesan – pesan berupa kesalahan pada jaringanMenampilkan display dari sebuah jaringan

Lapisan ini adalah lapisan yang paling dekat dengan user / pengguna, lapisan ini menjalankan aplikasi-aplikasi untuk user, menyediakan layanan jaringan untuk aplikasi user.

Layer Application (layer pertama pada pengirim email, dan layer terakhir pada penerima email), Penggunaan web browser untuk mengirimkan / menerima email (seperti IE, Chrome, Firefox)

23. OSI memiliki beberapa layer ,sebutkan dan jelaskan masing masing dari layer tersebut


Jawaban:

1. Application Layer (Lapisan ke-7)

Application layer adalah lapisan yang menjadi pusat (center) terjadinya suatu interaksi antara pengguna (end user) dengan aplikasi yang bekerja menggunakan fungsionalitas sebuah jaringan. Selain itu juga mempunyai fungsi untuk melakukan konfigurasi mengenai bagaimana cara aplikasi dapat bekerja menggunakan resource jaringan.

2. Presentation Layer (Lapisan ke-6)

Lapisan yang keenam adalah presentation layer, dimana mempunyai fungsi untuk mentranslasikan format data yang akan ditransmisikan oleh aplikasi melalui jaringan, ke dalam format yang dapat ditransmisikan oleh sebuah jaringan.

Pada layer ini, data juga akan ter- enkripsi dan dekripsi melalui sistem. Contoh protokol yang berada pada presentation layer adalah MIME, SSL, TLS, dan lain sebagainya.

3. Session Layer (Lapisan ke-5)

Session layer merupakan lapisan yang berfungsi untuk mendefinisikan bagaimana sebuah koneksi dapat dibuat, dikelola, dan dikembangkan. Contoh protokol yang berada pada session layer adalah NFS, SMB, RTP, dan lain – lain.

4. Transport Layer (Lapisan ke-4)

Transport layer mempunyai fungsi untuk memecah data menjadi paket – paket data, serta memberikan nomor urut untuk setiap paketnya. Sehingga, nantinya dapat disusun kembali saat sampai pada tujuan. Pada layer ini juga menentukan protokol yang akan digunakan untuk mentransmisikan data, seperti protokol TCP.

Protokol tersebut akan mengirimkan paket data, sekaligus memastikan bahwa setiap paket telah diterima dengan sukses dan tepat sasaran. Selain itu, juga dapat mentransmisikan ulang terhadap paket yang hilang atau rusak ketika proses pengiriman.

5. Network Layer (Lapisan ke-3)

Tugas dari network layer adalah membuat header untuk paket yang berisi informasi IP (Internet Protocol), baik IP pengirim atau IP tujuan data. Pada suatu kondisi, network layer juga melakukan proses routing melalui internetworking dengan menggunakan bantuan router dan switch pada layer ke-3.

6. Data-Link Layer (Lapisan ke-2)

Pada data-link layer memiliki tugas untuk menentukan setiap bit data dikelompokkan menjadi format yang disebut dengan frame. Pada level ini juga terjadi koreksi kesalahan, flow control, pengalamatan hardware atau perangkat keras (seperti halnya pada MAC Address (Media Access Control Address)).

Serta, menentukan bagaimana perangkat jaringan seperti hub, repeater, bridge, dan switch pada layer 2 dapat beroperasi. Untuk spesifikasi IEEE 802, dapat membagi tingkatan menjadi 2 level, yaitu lapisan Media Access Control (MAC) dan lapisan Logical Link Control (LLC).

7. Physical Layer (Lapisan ke-1)

Dan model OSI Layer terakhir dan yang paling utama adalah physical layer. Fungsinya adalah untuk mendefinisikan media transmisi jaringan, sinkronisasi bit, metode pensinyalan, serta membangun arsitektur jaringan seperti pengkabelan dan topologi jaringan.

Pada tahapan atau level ini juga mendefinisikan mengenai bagaimana sebuah NIC (Network Interface Card) dapat berinteraksi secara langsung dengan media kabel dan perangkat radio. Untuk setiap pengiriman data melalui tiap layer, dapat dianalogikan seperti anda mengirim surat.

28. Disebut apakah istilah dari sebuah data saat proses encapsulasi pada Layer 3 di OSI Layer


Jawaban:

Pada Layer 3 di OSI Layer, data yang dikelompokkan dalam sebuah unit informasi disebut "datagram" atau "paket".

Penjelasan:

Encapsulasi adalah proses menambahkan informasi tambahan ke dalam datagram atau paket, seperti informasi tambahan tentang jalur yang harus ditempuh atau informasi tentang tujuan akhir datagram atau paket tersebut. Encapsulasi ini dilakukan untuk membantu datagram atau paket tersebut sampai ke tujuannya dengan benar.
